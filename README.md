
## About
Matching the right dart SDK snapshot hash with the right Engine commit is the first step in Reverse engineering Dart/Flutter projects. There are sources online where you can find these data (e.g [reFlutter Repo](https://github.com/Impact-I/reFlutter/blob/main/enginehash.csv), [Darter Repo](https://github.com/mildsunrise/darter/blob/master/info/versions.md#flutter-version-table)).  However, I tried to get the hash for a Flutter app built on Flutter 2.3.3 (Dart version 2.13.4), but I didn't find any matching for that hash. The snapshot hash is calculated via [make_version.py](https://github.com/dart-lang/sdk/blob/main/tools/make_version.py) script. However, the script was broken in some versions, such as Flutter 2.3.3. As a result, there was no online snapshot hash for that version because the script would return nothing! I wrote a script to fix [make_version.py](https://github.com/dart-lang/sdk/blob/main/tools/make_version.py) in those versions, and here's the complete list ;)
